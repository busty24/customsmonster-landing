FAQ

Common questions

What is Customs Monster?+
Customs Monster is EU customs declaration software for customs brokers. It reads your shipping documents, extracts every relevant field, classifies goods to TARIC codes with confidence scores and written explanations, and outputs EDS-ready XML or JSON for submission to your national customs portal. It handles the data work. Your brokers handle the review.
Which document formats are supported?+
PDF, Excel (.xls, .xlsx, .xlsm, .xlsb), OpenDocument (.odf, .ods, .odt), images (.jpg, .jpeg, .png, .tiff), and XML. No reformatting or templates required. Upload documents in whatever format you receive them, including documents in any language.
How does the TARIC classification work?+
The system reads product descriptions from all uploaded documents, cross-references the EU TARIC tariff database, and returns the most appropriate code with a confidence score and a written explanation of why the heading applies. Brokers accept, reject, or override any result. Each decision trains the knowledge base for future declarations involving similar products.
What happens when documents have conflicting data?+
When two documents carry different values for the same field, the conflict is flagged with both values shown alongside their source documents. The broker selects the correct value before the declaration is finalised. This catches the discrepancies that most commonly cause customs rejection. The issue is visible before submission, not after.
Which EU countries and customs systems does it support?+
Currently integrated with Latvia's EDS (Electronic Declaration System) for both import and export declarations. Output is XML for exports and JSON for imports. The platform is built for expansion to additional EU member state systems. Contact us to discuss your country's requirements.
What trade corridors and product types has it been tested on?+
Customs Monster is live in production handling EU trade corridors including Japan, Germany, Brazil, and the USA. It has been tested on industrial equipment, precision electronics, raw materials, and consumer goods. The AI reads supplier documents in any language.
Is the data GDPR compliant?+
The platform operates within EU infrastructure. Your shipping documents and declaration data are not used to train AI models. Data is fully isolated per organisation account. GDPR compliance is a primary design constraint, not a checklist item.
Is it compliant with EU Union Customs Code requirements?+
Yes. Customs Monster is built around the requirements of the EU Union Customs Code (UCC) and the TARIC integrated tariff for both import and export declarations. Output files are formatted for submission to national EU customs filing portals.
